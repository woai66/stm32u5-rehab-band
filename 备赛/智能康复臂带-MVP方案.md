# AI-RehabBand 智能康复臂带 MVP 方案

## 项目定位

本版本以一个月左右的开发周期为边界，目标是做出一个能够稳定演示的训练评估与反馈型康复原型。系统以双 IMU 为运动感知主体，加入简化肌电采集，用于反映患者主动参与情况，在 STM32 平台上完成动作识别、角度计算、参与度评估、异常提醒和结果展示。

## 一、MVP 阶段解决的问题

脑卒中上肢康复训练中，常见问题并不只是动作有没有做出来，还包括动作是否标准、是否存在主动发力、训练过程是否安全可控。MVP 阶段聚焦以下四项能力：

1. 实时采集上臂与手腕运动数据。
2. 识别少量高相关上肢康复动作。
3. 计算肘关节角度、次数和基础训练评分。
4. 通过简化肌电信号判断主动参与程度。

## 二、MVP 功能边界

### 当前保留的功能

- 双 IMU 采集上臂和手腕姿态数据。
- 1 组简化肌电采集，用于主动参与判断。
- STM32 本地完成数据处理与训练逻辑。
- 实时计算肘关节角度。
- 识别 4 类基础上肢动作。
- 输出完成度、平稳度、参与度和综合评分。
- 提供训练引导、主动参与、异常提醒三种模式。
- 通过 OLED 和蓝牙输出训练结果。

### 当前不纳入 MVP 的功能

- 气动助力或被动牵引执行机构。
- 多通道肌电阵列。
- 压力阵列。
- 云端平台和医生端 Web。
- 临床量表映射。
- 复杂个性化处方系统。

当前阶段采用的策略是，在保留作品专业度的同时，把执行复杂度控制在可以按时完成的范围内。

## 三、现场演示目标

比赛现场应完成如下演示流程：

1. 用户佩戴上臂和手腕传感器，贴附肌电电极。
2. 设备开机，进入训练模式选择界面。
3. 用户选择指定动作并开始训练。
4. 系统实时显示当前动作、肘关节角度、肌电参与状态、动作次数和当前分数。
5. 训练结束后在本机显示本次结果，并通过蓝牙发送到手机端。

通过这个流程，作品能够形成完整的训练闭环，而不是停留在单纯的数据采集层面。

## 四、推荐硬件组成

| 模块 | 型号建议 | 数量 | 用途 |
|------|----------|------|------|
| 主控 | STM32U575 开发板 | 1 | 数据处理、算法运行、显示与通信 |
| IMU 传感器 | 逐飞 IMU660RB（核心芯片 LSM6DSRTR） | 2 | 上臂与手腕姿态采集 |
| 肌电采集 | MyoWare 2.0 / ADS1292R 简化方案 | 1 组 | 主动参与检测 |
| 显示屏 | 0.96 寸 OLED（驱动IC是ST7789P3） | 1 | 显示动作、角度、状态与评分 |
| 蓝牙模块 | HM-10 / ESP32-C3 串口透传 | 1 | 手机通信 |
| 电池 | 500mAh 锂电池 | 1 | 便携供电 |
| 固定结构 | 臂带、表带、电极贴片、小外壳 | 1 套 | 佩戴与封装 |

### 选型说明

- 单主控架构更适合一个月周期开发。
- 双 IMU 仍然是作品的核心价值，用于关节相对运动分析。
- IMU 优先采用逐飞 `IMU660RB` 成品模块，减少底层硬件焊接和传感器调试成本。
- 肌电只做简化主动参与检测，不做复杂医学级分析。
- OLED 足够承载现场展示信息，开发成本低。

## 五、系统架构

```text
上臂 IMU ─┐
手腕 IMU ─┼─> STM32U575 ─> 姿态解算/动作识别 ─> 评分计算 ─> OLED
肌电采集 ─┘                                           │
                                                     └─> 蓝牙发送到手机
```

### 数据流

```text
传感器采集 -> 数据滤波 -> 姿态解算 -> 相对姿态求角度 -> 肌电参与判定 -> 动作识别 -> 评分输出
```

## 六、软件功能拆解

### 1. 传感器采集

- IMU 采样频率：50Hz-100Hz。
- 肌电采样频率：建议 200Hz-500Hz，MVP 可先做包络或阈值判定。
- 采集内容：
  - 上臂 IMU：ax, ay, az, gx, gy, gz
  - 手腕 IMU：ax, ay, az, gx, gy, gz
  - 肌电信号：emg_raw / emg_level

### 2. 姿态解算与角度计算

- 使用 Madgwick 或互补滤波进行姿态估计。
- 结合上下肢相对姿态计算肘关节近似角度。
- 提供动作幅度、平均角度、最大角度等训练指标。

### 3. 动作识别

MVP 只保留最适合展示、最容易做稳的 4 类动作：

| 编号 | 动作 | 主要依据 |
|------|------|----------|
| 0 | 肘关节弯曲 | 关节角度周期性变化 |
| 1 | 手臂外展 | 上臂姿态抬升明显 |
| 2 | 前臂转动 | 陀螺仪角速度变化明显 |
| 3 | 肩上举 | 上臂大幅上抬 |

### 4. 肌电参与判定

MVP 阶段不追求复杂肌电分类，只完成以下能力：

- 对肌电原始信号做整流和平滑。
- 提取简单幅值特征或 RMS 值。
- 根据阈值判断用户是否存在主动发力。
- 将主动参与结果映射到训练评分中。

### 5. 评分机制

建议采用可解释的基础评分：

```text
总分 = 完成度 * 0.5 + 平稳度 * 0.25 + 节奏达标 * 0.1 + 主动参与度 * 0.15
```

评分项说明：

- 完成度：动作幅度是否达到目标区间。
- 平稳度：训练过程中抖动和突变是否过大。
- 节奏达标：动作频率是否接近预设节奏。
- 主动参与度：训练时是否出现有效肌电激活。

### 6. 本地显示与手机端同步

OLED 推荐显示内容：

- 当前模式
- 当前动作
- 实时角度
- EMG 状态
- 当前次数
- 当前评分

手机端推荐保留：

1. 设备连接页
2. 本次训练结果页
3. 历史训练记录页

## 七、训练模式设计

### 模式一：引导训练模式

- 适用场景：动作学习、重复训练。
- 功能：给出目标动作、节奏和目标角度范围。
- 输出：次数、达标率、平均角度、平均分数。

### 模式二：主动参与模式

- 适用场景：恢复期训练。
- 功能：在动作检测基础上叠加肌电参与判定。
- 输出：主动参与次数、参与度评分、综合训练评分。

### 模式三：异常提醒模式

- 适用场景：动作速度过快、抖动过大、角度超限。
- 功能：检测异常角速度或异常姿态变化，并提示减速或暂停。
- 输出：异常次数、异常类型、训练中断提醒。

## 八、算法实现策略

### 第一阶段：规则法优先

在一个月周期内，优先使用规则法和阈值法：

- 肘关节弯曲：角度在阈值区间内往复变化。
- 手臂外展：上臂抬升角度超过阈值。
- 前臂转动：特定轴角速度呈往复变化。
- 主动参与判定：EMG 包络值超过阈值并维持一定时间。

### 第二阶段：轻量 AI 预留

如果前述部分提前稳定，可补充轻量模型：

- 输入：1 秒窗口特征。
- 特征：IMU 时域特征 + 简化 EMG 幅值特征。
- 模型：小型全连接网络或 1D CNN。
- 目标：模型体积控制在 50KB 左右。

## 九、验收指标

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 动作识别准确率 | >= 85% | 4 类基础动作 |
| 主动参与判定成功率 | >= 80% | 基于阈值法 |
| 角度测量重复性误差 | <= 5 deg | 同一动作多次测量 |
| 系统响应延迟 | <= 300ms | 从动作发生到屏幕刷新 |
| 连续运行时间 | >= 2h | 实测结果 |
| 蓝牙传输成功率 | >= 95% | 训练结果稳定发送 |

建议至少准备以下实验结果：

1. 动作识别结果统计表。
2. 肘关节角度变化曲线。
3. EMG 激活示意曲线。
4. 单次训练结果示例。

## 十、4 周开发安排

### 第 1 周

- 打通双 IMU 通信。
- 完成简化肌电采集验证。
- 完成 OLED 显示和蓝牙数据发送。

### 第 2 周

- 完成姿态解算。
- 完成肘关节角度计算。
- 完成 2-3 类动作规则识别。
- 完成基础肌电阈值判定。

### 第 3 周

- 完成评分机制。
- 完成训练模式切换逻辑。
- 完成手机端基础展示。
- 开始采集测试数据。

### 第 4 周

- 完成联调。
- 进行准确率、延迟、续航和参与判定测试。
- 修复关键误判。
- 整理答辩材料和演示视频。

## 十一、人员分工建议

| A | STM32 驱动、IMU/EMG 采集、姿态解算 |
| B | 动作识别、EMG 判定、评分算法、测试数据整理 |
| C | OLED 界面、蓝牙通信、手机端展示、文档和答辩材料 |

## 十二、当前版本的创新点定义

1. 双 IMU 相对姿态测角，用于上肢关节运动评估。
2. 姿态信息与简化肌电参与信号结合，用于反映主动参与程度。
3. 本地实时训练评估与异常提醒，不依赖云端完成核心闭环。

## 十三、结论

当前版本更适合定位为训练评估与反馈型智能康复臂带。通过双 IMU 和简化肌电采集，系统能够在有限周期内完成动作识别、角度计算、主动参与评估和异常提醒，既避免了执行助力设备的高风险，也保留了康复场景下较强的专业性和展示价值。
